BUG REPORT
═══════════════════════════════════════════════════════════════════════════════

TITLE: Animation de réorganisation non visible

DESCRIPTION:
Lors du tri des équipes et joueurs par rapidité de réponse (v2.44.1), les
animations de réorganisation ne sont pas visibles. Les joueurs changent de
position instantanément au lieu de glisser en douceur.

SEVERITY: Medium (feature logically works, but lacks visual feedback)

ROOT CAUSE:
CSS overflow: hidden sur .team-card et .team-card-header créait un stacking
context qui clippait les CSS transforms générées par framer-motion.

TECHNICAL EXPLANATION
═══════════════════════════════════════════════════════════════════════════════

How the bug occurred:
  1. Feature v2.44.1 added layout animations with framer-motion
  2. framer-motion uses CSS transforms to animate position changes
  3. CSS overflow: hidden created a stacking context
  4. Stacking context clips content outside its bounds
  5. Transforms applied but visual result was clipped
  6. User saw instant position changes (animations invisible)

Why the fix works:
  1. Changed overflow: hidden to overflow: visible
  2. Removes stacking context creation
  3. CSS transforms now visible
  4. framer-motion animations render correctly
  5. Text clipping handled by text-overflow: ellipsis on .team-name
  6. No side effects on layout
